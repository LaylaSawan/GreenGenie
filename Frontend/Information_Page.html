Why Forest Sense is Needed

Forests are essential for biodiversity, climate regulation, and ecological balance. Forest Sense is a system that uses a mesh network of nodes equipped with sensors to detect forest fires. By analyzing real-time pictures captured by camera nodes, Forest Sense can detect smoke or flames and alert the relevant authorities, helping to protect our forests and the planet for future generations.

What Hardware Materials Did Forest Sense Use?

In order to create the Forest Sense working prototype, I used four ESP-32s, one ESP-32-CAM, two MQ-135 sensors, two TMP36 temperature sensors, two Humidity sensors, and two Flame sensors. Additionally, I used JavaScript, MySQL, HTML, and CSS to build the website and backend. For the software development, testing, and system integration, I relied on the Arduino IDE, VS Code, CPanel, and Amazon Web Services (AWS).

What Are The Challenges That Forest Sense Faced?

Throughout the Forest Sense project, I encountered several challenges due to the complexity of the system and the science. The hardest challenges that helped me learn and grow include: Finding the precise values from the sensors: Having to do trial and error to make sure that the values during the fire are precise and not insert any false values into the database. Connecting Front-end to Database: It was challenging to connect the front-end to the database as I had to figure out how to get the image link of my ESP-32-CAM uploading and correctly displayed on the interface. Unlike last year, where I only used AWS, I had to work with multiple services.

Why I added a mesh network

A mesh network topology was used to connect Forest Sense nodes. This topology enhanced network reliability and provided redundancy, so that if one node fails, data can still be routed through alternative paths. Additionally, mesh networks make it easier to add new nodes to the Forest Sense network.
